iStyle 1.20

Fast and Small Automatic Formatter
for Verilog Source Code

By haimag, (Report Bugs: haimag@gmail.com) 

To use from the command line:

Usage  :   iStyle [options] OriginalSourceFile  [...]

example:  iStyle --style=ansi Foo.v  ../Bar.v

            or

               iStyle   --style=gnu  F*.v  *b.v           

            or

               iStyle  --options=optiFile --style=kr  Foo.v

Options:

By default, no options input, with 4 spaces perindent, a maximal indentation of 40 spaces inside continuous statements, and NO formatting

 

Options may be written in two different ways:

 

1.      Long options:

These options start with '--', and must be written one at a time
(e.g. --brackets=attach  --pad  --indent=spaces=4 ).

2.      Short Options:

These options start with a single '-', and may be appended together.
Thus, writing -bps4 is the same as writing -b -p -s4 .

 

A default options file may be used to define default options:

 

·    iStyle looks for the default options file in the following order:
1. The value of the the system variable ISTYLE_OPTIONS if one exists.
2. The file named . iStylerc in the directory pointed to by the HOME system variable (i.e. $PATH/. iStylerc)
3. The file named . iStylerc in the directory pointed to by the HOMEPATH system variable (i.e. %HOMEPATH%\. iStylerc)

·   Options may be set apart by new-lines, tabs or spaces.

·   Long options may be written in the options file without the preceding '--'.

·   Lines within the options file that begin with '#' are considered line-comments.

·   Example contents of a default options file:

# default parsing is of gnu files
style=gnu
# Insert space paddings around parenthesies only
pad=paren
# Insert space paddings around operators only
pad=oper
#Insert empty lines around unrelated blocks
break-blocks
# suffix of original files should be .pre
suffix=.old

The following indentation options are currently available:

 

    

options

description

example

indentation options

-s#  or

--indent=spaces=#

Indent using # spaces per indent (e.g.-s4 or --indent=spaces=4 ).

 
 

-t   or

-t#   or 

--indent=tab=#

Indent using tab characters. Treat each tab as # spaces. If no '#' is set, treats tabs as 4 spaces.

 
 
 
-T#  or
--force-indent=tab=#

 

Indent using tab characters. Treat each tab as # spaces. Uses tabs as indents in areas '--indent=tab' prefers to use spaces, such as inside multi-line statements.

 
 

-B or

--indent-brackets

Add extra indentation to brackets.

 
The default:
if (SYN_flag) 
begin 
   Syn = 1'b0; 
end 
becomes: 
if (SYN_flag) 
    begin 
    Syn = 1'b0; 
    end

-G or

--indent-blocks

Add extra indentation to entire blocks.

 
The default:
if (SYN_flag) 
begin 
    bar = 1'b0; 
end  
becomes:
if (SYN_flag) 
    begin 
       bar = 1'b0; 
    end 
-M# or 
--max-instatement-indent=#

 

Indent a maximal # spaces in a continuous statement, relatively to the previous line (e.g. --max-instatement-indent=40)

 
 
 
-m# or -m
--min-conditional-indent=#

 

Set the minimal indent that is added when a header is built of multiple-lines. This indent makes helps to easily separate the header from the command statements that follow. The default setting for this option is twice the current indent. If no '#' is set, treats as 0 (e.g. --min-conditional-indent=8)

 
The default:
if (a < b
        || c > d)
   Syn = 1'b0;
When set #=0
if (a < b
    || c > d)
    Syn = 1'b0;
 
--indent-preprocessor

Indent multi-line #define statements

 
 
 
-E or 
--fill-empty-lines

Fill empty lines with the white space of their previous lines.

 
 

 

The following formatting options are currently available:

 

options

description

example

 
formatting options
 
 
-b or

--brackets=break

Break brackets  from their pre-block statements

 
if (SYN_flag) 
begin 
    Syn = 1'b0; 
end 
else 
begin 
    Syn = 1'b1; 
end
 

 

-a or

--brackets=attach

Attach brackets to their pre-block statements ( i.e. K&R style )

 
if (SYN_flag)begin 
    Syn = 1'b0; 
end 
else begin 
    Syn = 1'b1; 
end
 

 

--break-blocks

Pad empty lines around header blocks (e.g. 'if', 'while'...)

 
SYN_flag = 1'b1;
if (SYN_flag) begin 
    Syn = 1'b0; 
end 
else begin
    Syn = 1'b1; 
end
becomes: 
SYN_flag = 1'b1;
 
if (SYN_flag) begin 
    Syn = 1'b0; 
end 
else begin
    Syn = 1'b1; 
end 
 

 

--break-blocks=all

Pad empty lines around header blocks (e.g. 'if', 'while'...). Treat closing header blocks

 
SYN_flag = 1'b1;
if (SYN_flag) begin 
    Syn = 1'b0; 
end 
else begin
    Syn = 1'b1; 
end
becomes: 
SYN_flag = 1'b1;
 
if (SYN_flag) begin 
    Syn = 1'b0; 
end 
 
else begin
    Syn = 1'b1; 
end 
 

 

--break-elseifs

Break 'else if()' header combinations into seperate lines.

 
if (SYN_flag) begin 
    Syn = 1'b0; 
end 
else if (isSyn)
begin
    Syn = 1'b1; 
end
becomes: 
if (SYN_flag) begin
    Syn = 1'b0; 
end 
else 
    if (isSyn)begin
        Syn = 1'b1;
    end

 

-p or

--pad=oper

Insert space padding around operators only.

 
if (SYN_flag) 
    a = ((b-c)*a); 
becomes: 
if (SYN_flag) 
    a = ((b - c) * a);
 

 

--pad=paren

Insert space padding around parenthesies only

 
if (SYN_flag) 
    a = ((b-c)*a+d); 
becomes: 
if ( SYN_flag ) 
    a = ( ( b-c )*a+d );
 

-l

or

--pad=block

 

 

Enclose one statement in a begin-end only for keyword “if / else / while /for”

 
if (SYN_flag) 
    Syn = 1'b0; 
else if (isSyn)
    Syn = 1'b1; 
becomes: 
if (SYN_flag)
begin 
    Syn = 1'b0; 
end 
else if (isSyn)
begin
    Syn = 1'b1; 
end
 
 
-P or 
--pad=all
 

Insert space padding around operators AND parenthesies

 
if (SYN_flag) 
   a = ((b-c)*a); 
becomes: 
if ( SYN_flag ) 
    a = ( ( b - c ) * a );
 
 
 
-o or
--one-line=keep-statements

Don't break complex statements and multiple statements residing in a single line

 
if (SYN_flag)
begin  
    SYN_flag = 1'b0; a=b;
end
remains as is. 
 
if (SYN_flag) DoSyn = 1'b0; 
remains as is.
 
-O or
--one-line=keep-blocks

Don't break one-line blocks.

 
if (SYN_flag)
begin SYN_flag = 1'b0; end
remains as is.

 

 

The following other options are currently available:

 

--suffix=####
Append the suffix #### instead of '.orig' to original filename. (e.g. --suffix=.old)

-n or --suffix=none

Tells Astyle not to keep backups of the original source files.

WARNING: Use this option with care, as Astyle comes with NO WARRANTY...

-X or --errors-to-standard-output
Print errors and help information to standard-output rather than to standard-error.
This option should be helpful for systems/shells that do not have this option, such as in Windows-95.

-v or --version
Print version number.

-h or -? or --help
Print a help message

--options=####  OR --options=none

Parse used the specified options file: ####, options=none, none

parse options file, and not looks for parse options files
